# Obsidian Ingest Studio 使用说明

## 1. 安装

从 GitHub Releases 下载 `v0.04.2` 的 Windows 安装包并安装。

普通用户不需要安装 Python、Node.js、Rust 或 Visual Studio。软件运行账号登录时会使用 Windows 自带的 Microsoft Edge。

视频处理需要外部工具。软件内“依赖”页可以自动下载和配置 `yt-dlp`、`ffmpeg`：

| 功能 | 工具 |
| --- | --- |
| B站、YouTube、TikTok 下载 | `yt-dlp`、`ffmpeg`（可自动配置） |
| 抖音下载 | `douyin-dl`、`ffmpeg` |
| 音视频转文字 | `whisper` 或 `FunASR` |

`douyin-dl`、`whisper`、`FunASR` 仍按提示手动配置。只处理网页、RSS、本地文本和 PDF 时，可以先不安装这些媒体工具。

## 2. 首次配置

1. 打开 `Obsidian Ingest Studio`。
2. 在“配置”填写 Obsidian Vault 路径。
3. 填写 LLM Base URL、模型和 API Key。
4. 点击“保存”。
5. 打开“依赖”，点击“安装缺失工具”。
6. 点击“重新检测”或“健康检查”，确认 Vault、队列、LLM 和需要的媒体工具为 OK。
7. 在“输出方式”里确认是否同时生成 HTML、Excel/CSV 或 Notion。

你的 Vault 当前可填写：

```text
D:\obsidian资料汇总\HYL
```

## 3. 管理平台账号

打开一级导航“账号”。

支持的平台：

- 抖音
- 哔哩哔哩
- YouTube
- TikTok

添加账号：

1. 在对应平台点击“添加账号”。
2. 软件打开一个独立 Microsoft Edge 窗口。
3. 在平台网页完成登录和验证码。
4. 软件识别到昵称和平台 ID 后显示候选账号。
5. 点击“保存并切换”，或点击“仅保存”。

每个平台可以保存多个账号，但只有一个“当前”账号。采集时会自动使用该平台的当前账号。

账号操作：

- `切换`：把该账号设为当前账号。
- `校验`：检查登录态是否仍然有效。
- `重新登录`：打开新的独立 Edge 登录窗口，识别后再次确认。
- `删除`：删除该账号及其本机浏览器 profile。

密码、短信验证码和二次验证只在平台网页输入。软件不把密码写入配置、日志或 Git。

## 4. 处理抖音收藏

1. 打开“运行”。
2. 确认“抖音收藏”显示正确当前账号和“可用”。
3. 在“抖音数量”填写本次要处理的收藏数量。
4. 点击“处理抖音收藏”。
5. 软件先抓取收藏，再自动处理对应队列。

每次运行前都可以修改数量，不会自动定时执行。

## 5. 导入其他来源

打开“导入”。

- `链接`：每行一个链接，保存后点击“导入 links.txt”。
- `RSS`：每行一个 RSS / Atom 地址，填写数量后导入。
- `网页 URL`：粘贴文章链接后执行网页剪藏。
- `目录路径`：扫描本机目录中的 PDF、字幕、音频和视频。
- `平台列表`：选择 YouTube、B站或 TikTok，填写列表链接和数量。

页面顶部会显示 B站、YouTube、TikTok 当前账号。公开内容在没有账号时仍可尝试处理；需要登录的内容会使用当前账号。

## 6. 处理队列

打开“运行”，填写“处理数量”，点击“处理队列”。

打开“队列”可以：

- 查看待处理、处理中、失败、完成和跳过项。
- 重试单条任务。
- 批量重试失败任务。
- 跳过不再需要的任务。

## 7. 查看输出

处理完成后，Markdown 会写入配置中的 Obsidian Vault，并根据 DeepSeek 返回的分类写入允许的文件夹。

当前默认分类包括：

```text
AI学习
考研资料汇总
炒股与量化学习
研后/英语学习
工作
Life
Inbox/Learning Inbox
```

无法确定分类时会进入 `Inbox/Learning Inbox`。

配置页支持多种输出：

| 输出 | 默认位置 |
| --- | --- |
| Markdown / Obsidian | 配置中的 Vault 或普通文件夹 |
| HTML | `exports/html` |
| Excel / CSV | `exports/notes-index.csv` |
| Notion | 配置的 Notion Database |

配置页提供“打开 Markdown / 打开 HTML / 打开 CSV”按钮。第一次还没生成 HTML 或 CSV 时，软件会先创建对应目录或父目录。

## 8. 备份与恢复

配置页可以创建本地备份。备份会写到项目的 `backups/` 目录，文件名类似：

```text
obsidian-ingest-backup-20260628-120000.zip
```

备份包含：

```text
config.toml
data/queue.sqlite
accounts/
links.txt
feeds.txt
douyin-config.yml
.cookies.json
```

恢复时在配置页粘贴备份 zip 路径并确认覆盖。备份可能包含 API Key、Cookie 和账号登录态，只适合自用迁移，不要公开分享。

备份会排除浏览器缓存、Crashpad 指标、GPU/Shader 缓存和 Edge 保存密码数据库，避免包体过大或混入不必要的运行时数据。

命令行也可以使用：

```powershell
.\run.ps1 backup --json --config .\config.toml
.\run.ps1 restore .\backups\obsidian-ingest-backup-xxxx.zip --yes --json --config .\config.toml
```

## 9. 常见问题

### 账号显示“已失效”

在“账号”页点击“重新登录”，完成平台网页登录，再确认候选账号。

### 视频只生成占位笔记

打开“依赖”，先点击“安装缺失工具”配置 `yt-dlp` 和 `ffmpeg`。如果是抖音或音视频转文字，再检查 `douyin-dl`、`whisper` / `FunASR` 路径，然后运行健康检查。

### DeepSeek 没有生成摘要

检查 API Key、Base URL 和模型名称。API Key 留空保存时，软件会保留原有 Key，不会把已保存密钥显示出来。

### 找不到生成的笔记

检查：

1. “队列”中任务是否完成。
2. “配置”的 Vault 路径是否正确。
3. `Inbox/Learning Inbox` 和其它允许分类目录。
4. “日志”页面中的最新错误。

## 10. 本机数据位置

安装版默认工作区：

```text
%LOCALAPPDATA%\Obsidian Ingest Studio
```

其中：

```text
config.toml        配置
accounts/          账号元数据和独立 Edge profile
data/              队列数据库
cache/             下载与处理缓存
logs/              日志
inbox/             本地批量导入目录
```

`accounts/`、`config.toml` 和 API Key 都是本机私有数据，不应上传到 GitHub。
