# 本地稳定性验收

## 自动检查

每次准备给别人安装包前，先运行：

```powershell
cd D:\obsidian-ingest-pipeline
.\packaging\local-stability-check.ps1
```

这个脚本只使用本机临时目录，不访问真实平台、不使用云端模型。它会检查：

- 健康检查命令能运行。
- 20 个本地 Markdown 文件能入队并处理。
- 20 条链接能批量入队。
- 诊断包能导出。
- 备份包能创建。

## 人工检查

自动检查通过后，再用真实账号做小批量人工验证：

| 场景 | 建议数量 | 通过标准 |
| --- | ---: | --- |
| 抖音收藏 | 5-20 条 | 能拉取当前账号可访问内容；失败时提示登录态或风控原因 |
| B 站/YouTube 链接 | 5-20 条 | 能下载或入队；下载器失败时提示依赖或平台错误 |
| PDF/本地文件 | 10 个 | 能生成 Markdown；无法解析时进入失败队列并给出原因 |
| 普通网页 | 10 个 | 能提取标题和正文；无法访问时不影响其它任务 |
| 错误 API Key | 1 次 | 提示到配置页重新填写 |
| 输出目录无权限 | 1 次 | 提示更换输出目录或权限 |

## 不做自动化的平台压测

真实平台压测容易触发验证码、限流或账号风险，不适合放进自动脚本。对外交付时，应把平台导入描述为“支持用户在已登录状态下导入可访问内容”，不要承诺永久稳定抓取全部收藏。
